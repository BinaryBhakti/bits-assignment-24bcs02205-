Library Manager Project Report

Entity Relationship Design

The application manages two entities: Author and Book. One author can write many books, while every book belongs to exactly one author.

Author: id, name, email, country, birthYear

Book: id, title, genre, publishedYear, isbn, author

Relationship: Author @OneToMany Book and Book @ManyToOne Author

JPA creates the authors and books tables. The books table contains the foreign key author_id.

Populate Database

The DataSeeder class implements CommandLineRunner. At application startup, it inserts 10 authors and 10 books if the database is empty. Hibernate creates the schema automatically from JPA annotations using spring.jpa.hibernate.ddl-auto=create.

Create Operation

Create forms are implemented in JSP pages:

  • src/main/webapp/WEB-INF/jsp/authors/form.jsp
  • src/main/webapp/WEB-INF/jsp/books/form.jsp

Controllers handle submissions through POST /authors and POST /books. Submitted objects are validated with Jakarta Bean Validation annotations. DataIntegrityViolationException is caught to show friendly duplicate-value errors for unique fields such as email, title, and ISBN.

Read Operation

List pages are implemented in JSP:

  • src/main/webapp/WEB-INF/jsp/authors/list.jsp
  • src/main/webapp/WEB-INF/jsp/books/list.jsp

The book list calls the service layer, which uses the repository method findBookAuthorDetails(). That repository method performs an inner join between Book and Author and returns a DTO projection named BookAuthorView.

Update Operation

Edit links are available on the list pages. The update forms reuse the creation JSP files. Controllers load existing records with GET /authors/{id}/edit and GET /books/{id}/edit, then persist changes with POST /authors/{id} and POST /books/{id}.

Testing

Repository testing uses @DataJpaTest to verify the custom inner join query. Service testing uses JUnit 5 and Mockito to verify business logic and repository interaction.

mvn test

Challenges Faced

The main challenge was cleanly binding a JSP form to a related entity. The book form submits the selected author through author.id; the service loads the managed Author before saving the Book. This keeps persistence behavior stable and avoids detached entity problems.
